3 Kiến Trúc Hệ Thống

Hệ thống được tổ chức theo mô hình Monorepo — một Express server duy nhất phục vụ cả API và toàn bộ file HTML tĩnh của frontend. Kiến trúc đơn giản nhưng đủ mạnh cho quy mô đồ án và dễ dàng tách ra microservice khi cần scale.

4 Các Tính Năng Đã Triển Khai
4.1 Quản Lý Người Dùng & Xác Thực
✅
Đăng ký / Đăng nhập / Đăng xuất
JWT lưu trong HTTP-only cookie (authToken), bảo vệ khỏi XSS. Validate email regex, mật khẩu tối thiểu 6 ký tự.
✅
Quên mật khẩu qua OTP Email
Gửi mã OTP qua Gmail App Password → xác nhận OTP → đặt lại mật khẩu. Xử lý trong authController.js.
✅
Upload avatar & đổi thông tin cá nhân
Multipart form upload, lưu vào backend/uploads/, cập nhật avatar_url trong DB.
✅
Phân quyền role-based (user / admin)
Middleware authMiddleware + requireAdmin bảo vệ tất cả API admin. Route HTML admin dùng authenticateHTML(['admin']).
4.2 Hệ Thống Quản Lý Truyện
✅
CRUD truyện đầy đủ
Thêm/sửa/xóa truyện. Xóa cascade toàn bộ chapters và chapter_contents liên quan. Sau mỗi thao tác đồng bộ ngay với Elasticsearch.
✅
Đọc truyện theo chương — Lazy Crawl + Cache
GET /api/chapters/:id/content: kiểm tra chapter_contents trước, nếu chưa có mới crawl và lưu cache vào DB. Hỗ trợ truyện tranh dạng ảnh.
✅
Ghi lịch sử xem
Mỗi lần user mở truyện ghi vào user_story_views, dùng cho gợi ý và thống kê truyện hot tuần.
✅
Đánh giá truyện (Rating 1–5 sao)
1 user chỉ được đánh giá 1 lần mỗi truyện (UNIQUE constraint). GET /api/rating trả về rating trung bình cùng tổng số lượt.
✅
Danh sách yêu thích (Favorite Lists)
User tạo nhiều danh sách, thêm/xóa truyện vào từng danh sách. Hỗ trợ tên danh sách tùy chỉnh.

4.5 Tóm Tắt Truyện Bằng AI (Groq)
✅
POST /api/ai/summarize — tóm tắt nội dung truyện
Model llama-3.1-8b-instant, temperature 0.3. Cache kết quả vào cột ai_summary trong bảng stories — gọi AI 1 lần duy nhất, các lần sau trả về ngay từ DB.
4.6 Gợi Ý Truyện Thông Minh
✅
Gợi ý dựa trên lịch sử đọc + điểm số
Phân tích user_story_views lấy genres yêu thích. Scoring: genre match 35% + view_count 20% + avg_rating 15%. Loại trừ truyện user đã đọc.
4.7 Hệ Thống Crawl Truyện
✅
Crawler trang tĩnh (Axios + Cheerio)
crawler.js và crawlALL.js xử lý các trang không cần JavaScript render. Batch crawl danh sách truyện.
✅
Crawler trang JS-rendered (Puppeteer fallback)
crawlChapterList.js: ưu tiên Axios + Cheerio, tự động fallback sang Puppeteer khi trang yêu cầu JavaScript render (SPA, infinite scroll).
✅
Lazy Crawl + Cache chương
Crawl chương chỉ khi user mở lần đầu. Lưu ảnh vào chapter_contents (JSONB). Lần sau đọc từ cache DB, không crawl lại.
4.8 Bình Luận & Tương Tác
✅
Hệ thống bình luận dạng cây (nested comments)
CRUD đầy đủ: tạo, sửa, xóa comment (kèm toàn bộ replies qua CTE). parent_id cho phép reply vô hạn cấp.
✅
Like / Unlike bình luận
Upsert vào comment_likes (UNIQUE user_id + comment_id). Thông báo realtime đến chủ comment qua Socket.io.
4.9 Quản Trị Hệ Thống (Admin Panel)
✅
Thống kê tổng quan
GET /api/stat: tổng số truyện, người dùng, lượt xem, bình luận, đánh giá. Truyện hot tuần qua user_story_views (7 ngày gần nhất).
✅
Quản lý người dùng
CRUD users: thêm/sửa/xóa tài khoản, đổi role. Bảo vệ bởi requireAdmin middleware.
✅
Hệ thống báo lỗi (Error Report)
User gửi báo lỗi kèm screenshot. Admin xem danh sách, lọc theo trạng thái (pending/fixing/done/ignored), phản hồi và thông báo cho user.
4.10 Thông Báo Realtime (Socket.io)
✅
Hệ thống thông báo real-time
User join room theo email. Server emit newNotification khi: reply comment, like comment, admin phản hồi báo lỗi. Đánh dấu đã đọc qua API.

6 Cơ Sở Dữ Liệu (PostgreSQL)
6.1 Schema Các Bảng
stories
id, title, author, description, url, cover_url, genres(text[]), status, view_count, ai_summary, created_at
chapters
id, story_id (FK→stories CASCADE), chapter_num(float8), title, source_url, created_at.
UNIQUE(story_id, chapter_num)
chapter_contents
chapter_id (PK+FK→chapters CASCADE), images(jsonb), crawled_at. 1-1 với chapters
users
id, email (UNIQUE), username, password, role, avatar_url, otp, otp_expires
comments
id, story_id, user_id, parent_id, content, likes, created_at, updated_at
comment_likes
id, user_id, comment_id
UNIQUE(user_id, comment_id)
ratings
id, story_id, user_id, rating
UNIQUE(story_id, user_id)
user_story_views
id, user_id, story_id, viewed_at
Dùng cho recommend + popular-week
notifications
id, user_email, message, is_read, created_at
favorite_lists
id, iduser, name, created_at
favorite_stories
id, list_id, story_id, added_at
UNIQUE(list_id, story_id)
reports
id, title, story_url, message, screenshot_path, user_email, status(pending/fixing/done/ignored), response, created_at
chat_messages
id, user_id (FK→users CASCADE), story_id (FK→stories CASCADE), role(user/assistant), content, created_at

Quan hệ chapter:

stories (1) ──< chapters (N) ──── chapter_contents (1)
               story_id FK          chapter_id PK+FK
               CASCADE DELETE       CASCADE DELETE

Đặc điểm:
  · chapter_num dùng float8 — hỗ trợ chương thập phân (vd: 10.5, 1.5)
  · chapter_contents.images là JSONB array ["url1", "url2", ...]
  · Lazy crawl: chỉ điền chapter_contents khi user mở chương

8 Quy Tắc & Tiêu Chuẩn Phát Triển
Bảo mật
  • Không hardcode credentials — tất cả qua process.env
  • Không lộ error.message ra client — dùng message chung
  • Admin API bắt buộc authMiddleware + requireAdmin
  • Parameterized query ($1, $2, ...) — không nối chuỗi SQL
  • JWT trong HTTP-only cookie — bảo vệ khỏi XSS
Tổ chức code
  • Route: validate → service → response (không xử lý business logic)
  • Services chứa logic nghiệp vụ tái sử dụng
  • Tên biến/hàm: camelCase; tên bảng/cột: snake_case
  • try/catch/finally + client.release() với pg Pool
  • HTTP status code đúng nghĩa (400/401/403/404/500)
Validate input
  • Email: regex /^[^\s@]+@[^\s@]+\.[^\s@]+$/
  • Password: tối thiểu 6 ký tự
  • Rating: số nguyên từ 1 đến 5
  • ID params: parseInt, kiểm tra > 0
  • Validate tại route boundary trước khi chạm DB
Response format
  • Thành công: { message, data }
  • Lỗi: { message }
  • Tạo mới: status 201
  • Log format: [module] context: err
  • Thông báo lỗi tiếng Việt có dấu
9 Định Hướng Deploy Production
Hạ tầng
  • Infrastructure-as-Code: docker-compose.yml cho PostgreSQL + ES
  • Stateless app — không lưu trạng thái trong bộ nhớ
  • Port qua process.env.PORT || 3000
  • Bind 0.0.0.0 (không hardcode localhost)
Bảo mật production
  • HTTPS qua reverse proxy (Nginx/Caddy)
  • Cookie secure: true khi NODE_ENV=production
  • CORS giới hạn origin cụ thể
  • Elasticsearch không public ra internet
Database
  • Migration script thay vì sequelize.sync()
  • Connection pool max phù hợp với DB server
  • Uploads lưu Docker volume hoặc S3/R2
  • backend/uploads/* trong .gitignore
Graceful Shutdown
  • Xử lý SIGTERM từ container orchestrator
  • Đóng connection pool + HTTP server sạch
  • Log có cấu trúc đủ trace lỗi không cần SSH
